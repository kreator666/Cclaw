# 海报生成模块

> 将喜剧内容转化为视觉海报。Cclaw 负责数据层（组装 design brief），canvas-design skill 负责视觉生成。

## 工作流程

```
用户请求 → 识别海报类型 → 收集信息 → 填充 brief 模板 → 调用 canvas-design → 输出 .png/.pdf
```

## 海报类型

### 通用海报

| 类型 | 尺寸 | 用途 | 关键要素 |
|------|------|------|---------|
| standup-poster | 1080×1920 | 抖音/B站封面 | 演员名、段子名、一句话文案 |
| comedy-show | 1920×1080 | 演出宣传 | 演出名、日期、场地、演员阵容 |
| social-card | 1200×630 | 社交分享卡 | 金句/段子、出处、作者 |

## 信息收集规则

**场景A — 用户刚用 cclaw 创作完内容后请求海报：**
- 自动从创作输出中提取：段子标题→海报主题，Hook金句→一句话文案
- 只需确认：日期/场地（若有演出）、风格偏好

**场景B — 用户单独请求海报（无上下文创作）：**
- 必须交互确认关键信息（见各 brief 模板的"内容数据"字段）
- 缺少关键信息时主动询问，不猜测

## Design Brief 模板

`briefs/` 目录下存放三种海报的 design brief 模板：

| 文件 | 用途 |
|------|------|
| `briefs/standup-poster.md` | 脱口秀个人海报（竖版） |
| `briefs/comedy-show.md` | 演出宣传海报（横版） |
| `briefs/social-card.md` | 社交媒体分享卡 |

## 配方

| 文件 | 用途 |
|------|------|
| `recipes/quote-card.md` | 从段子中提取金句生成分享卡 |
