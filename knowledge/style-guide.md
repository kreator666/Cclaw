# 喜剧风格体系 · Style Guide

> 风格是喜剧大类下的第二层维度。同一个体裁，可以有完全不同的讲法。
> 风格不是"题材"，而是"怎么讲"的调性选择。

---

## 设计原则

1. **风格是调性，不是内容** — 同一个话题（如"相亲"），可以用自嘲风格讲，也可以用讽刺风格讲
2. **案例带风格标签** — 每个案例文件顶部标记 `style:`，方便检索
3. **无法归类时 AI 推荐** — 新案例入库时，根据内容推荐最合适的风格
4. **创作时风格可选** — 用户可以指定风格，也可以让 AI 根据 prompt 推荐

## 风格索引（按大类）

### 脱口秀 Style Taxonomy

| 风格 | 核心特征 | 适合话题类型 |
|------|---------|------------|
| **自嘲式** | 讲自己的糗事、尴尬、失败，与自己和解 | 相亲、职场、身材、社交、成长 |
| **观察式** | 从日常细节切入，挖掘荒诞日常 | 通勤、外卖、网购、排队、社恐 |
| **观点输出式** | 有立场、有态度，直接对某现象开炮 | 彩礼、职场PUA、房价、社会事件 |
| **故事讲述式** | 有完整故事线，用叙事推动笑点 | 个人经历、童年、家人、特殊事件 |
| **隐喻讽刺式** | 用比喻/类比/技术术语解构现实 | AI、科技、行业、内卷、元叙事 |
| **热点评论式** | 紧跟新闻事件，快速反应，观点鲜明 | 热搜事件、明星八卦、政策新规 |
| **荒诞解构式** | 不讲道理，把日常推至荒谬，笑点来自无逻辑 | 任何话题的极致脑洞版 |
| **温暖治愈式** | 笑着笑着被打动，有情感落点 | 亲情、友情、与自己和解、失去 |

#### 脱口秀风格详解

**自嘲式**
把自己的糗事、失败、尴尬拿出来讲，是最容易引发共鸣的风格。笑点来自"观众发现自己没那么惨"或"原来我们都一样"。
- 典型结构：惨开场 → 更惨发展 → 自我接纳收尾
- 禁忌：过度卖惨不反转，只吐苦水不消化

**观察式**
从日常生活中找出观众熟悉但不常细想的细节，用"居然还能这样"制造笑点。
- 典型结构：日常场景描述 → 细节放大 → 荒诞结论
- 关键：细节要真实，不能编造，否则观众无感

**观点输出式**
对某个现象有明确立场，直接开炮，笑点来自"说得太对了"的认同感。
- 典型结构：现象描述 → 立场表态 → 极端化演绎
- 关键：态度要鲜明，不能骑墙

**故事讲述式**
有完整叙事线，用故事的意外结局制造笑点，类似微型单口小说。
- 典型结构：建立背景 → 推进叙事 → 出人意料的结局
- 关键：故事要有画面感，细节要具体

**隐喻讽刺式**
用比喻、技术术语、跨领域类比来解构某个话题，笑点来自"重新定义"。
- 典型结构：现实话题 → 技术/学术/商业术语包装 → 荒诞结论
- 适用：AI、科技、行业黑话、内卷等抽象话题

**热点评论式**
紧跟新闻事件，快速反应，用喜剧视角解读时事。
- 典型结构：事件概述 → 荒诞推演 → 讽刺落点
- 时效性：热点事件7天内最佳

**荒诞解构式**
完全不讲道理，把日常推至荒谬。笑点来自"怎么可能这样"的无厘头。
- 典型结构：日常话题 → 无逻辑升级 → 彻底荒诞
- 关键：表面无逻辑，内部有节奏感

**温暖治愈式**
有情感落点，笑着笑着被打动。不是煽情，是真诚。
- 典型结构：轻松开场 → 情感铺垫 → 温暖收尾
- 关键：真诚，不刻意煽情，落点要轻不要重

### 漫才 Style Taxonomy

| 风格 | 核心特征 | 典型场景 |
|------|---------|---------|
| **日常离谱吐槽** | 把日常生活中的小事放大到荒诞程度 | 等电梯、点外卖、排队、网购 |
| **认知偏差反差** | 利用常识和反常识的碰撞制造笑点 | 专业术语vs普通人、说明书陷阱 |
| **人物矛盾冲突** | 两个固定角色因性格差异产生冲突 | 夫妻、情侣、闺蜜、父子、主仆 |
| **脑洞设定无厘头** | 建立一个完全荒诞的世界观并严格执行 | 异世界、反乌托邦、平行宇宙日常 |
| **身份错位** | 角色处于不属于自己身份的位置引发冲突 | 老板当员工、医生当病人、教授当学生 |

### Sketch（素描喜剧）Style Taxonomy

| 风格 | 核心特征 | 典型结构 |
|------|---------|---------|
| **经典场景夸张** | 把一个日常场景推到极致夸张 | 餐厅/医院/银行/机场的极限版本 |
| **身份错位反差** | 人物身份和行为的错位制造冲突 | 老板比员工紧张、医生比病人慌 |
| **社会现象讽刺** | 用戏剧化方式讽刺某个社会现象 | 内卷、加班文化、社交媒体 |
| **前提先行荒诞** | 建立一个荒诞前提然后完全按照逻辑演绎 | "如果所有人说话都只能反着说" |
| **连续反转冲击** | 不断反转，每次都颠覆观众预期 | 多层次反转，逐级升级 |

### 日式短剧（コント）Style Taxonomy

| 风格 | 核心特征 | 典型结构 |
|------|---------|---------|
| **极端性格压顶** | 一个偏执/懦弱/自大/较真的人被日常逼到极限 | 便利店偏执狂、完美主义者 |
| **日常场景打破** | 正常日常突然被打破，人物反应层层升级 | 办公室/医院/便利店/家里 |
| **目标层层阻碍** | 明确目标+每个阻碍都比上一个更荒谬 | 买奶茶、点餐、问路 |
| **荒诞反转收尾** | 结尾彻底颠覆前面建立的一切 | 逻辑反转、定义反转 |
| **逻辑错位死磕** | 一方用逻辑，一方用"重新定义"，各说各的 | 再见吧青春之光风格 |
| **装傻者死不认错** | 装傻者用权力/话语权逃避责任，吐槽者被气死 | 谭爷爷装傻版 |

## 案例风格标签系统

每个案例文件顶部标注：

```
style: [主风格], [副风格（可选）]
mood: [情绪标签，可选]
tags: [关键词，可选]
```

**示例：**
```yaml
---
style: 自嘲式, 故事讲述式
mood: 自我和解
tags: 相亲, 社交, 成长
---
```

## 无法归类时的推荐逻辑

当新案例无法匹配已有风格时，按以下顺序推荐：

1. **读内容** — 提取关键词（糗事/日常/观点/故事/比喻/新闻/荒诞）
2. **匹配风格** — 对照各风格的核心特征，找到最接近的
3. **推荐选项** — 给出2-3个最可能的风格选项，让用户确认
4. **入库标签** — 用户确认后，在文件顶部添加 `style:` 标签

**推荐Prompt模板：**
> 这个案例最适合哪种风格？我推荐：A（理由）/ B（理由）/ 其他（请描述）。

## 创作时的风格推荐

在四问流程中，风格是**最后一个问题**（Q4之后），用户可以：
- 选择 AI 推荐的风格
- 指定自己的风格
- 让 AI 根据 prompt 自动推断（默认行为）

**自动推断逻辑：**
- prompt 含"糗事""我的经历""社死"→ 自嘲式
- prompt 含"我发现""居然""日常"→ 观察式
- prompt 含"我觉得""就是""凭什么"→ 观点输出式
- prompt 含"朋友说""有一次""小时候"→ 故事讲述式
- prompt 含"AI""科技""程序员""模型"→ 隐喻讽刺式
- prompt 含"最近""新闻""热搜""听说"→ 热点评论式
- prompt 含"怎么可能""荒唐""无厘头"→ 荒诞解构式
- prompt 含"笑着笑着""感动""温暖""治愈"→ 温暖治愈式

**匹配到多个风格时：** 融合为主风格+次风格，在案例读取时优先读主风格案例。
