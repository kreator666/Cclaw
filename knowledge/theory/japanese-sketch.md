# 日式短剧（コント）创作理论

> 整合自：《爆笑短剧入门》（元祖爆笑王）、《笑之大学：三谷幸喜谈创作》、《偽りなきコントの世界》（岩崎う大）

---

## ⚡ 极简版：死磕这6条核心理论

1. **角色永远第一**：性格贯穿全程，不崩人设
2. **目标明确**：每个角色都有一个要完成的事
3. **起承转合闭环**：5分钟也要有完整故事线
4. **认真的荒诞**：角色越认真，观众越想笑
5. **节奏铁律**：15秒小笑点，1分钟大爆点
6. **装傻+吐槽**：用行为装傻，用反应吐槽

---

## 一、コント三要素（必学核心）

### 1. 角色
- **必须有明确性格**：偏执 / 懦弱 / 自大 / 较真（选一，贯穿始终）
- 人设绝对不能崩——无论剧情多荒诞，角色性格必须从头到尾一致
- 笑点来自性格驱动行为，而不是硬加梗

### 2. 情境
- 日常场景被打破：办公室、医院、便利店、家里
- **"日常场景 + 极端人物"黄金组合**：把一个偏执狂、强迫症、老好人放进日常里，冲突自动产生

### 3. 目的
- 角色有一个明确目标：请假、看病、要钱、证明自己
- **所有笑点都来自「角色性格 + 目标受阻」**

---

## 二、装傻（ボケ）& 吐槽（ツッコミ）的短剧化用法

### 基本原则
- 不是站着互怼，而是**角色行为装傻 + 另一个角色反应吐槽**
- 装傻是动作、是情境，吐槽是反应、是观众视角

### 三人结构
```
1 主装傻 + 1 主吐槽 + 1 旁观者（强化冲突）
```
- 旁观者可以是：另一个极端性格的角色，或者是"正常人"视角
- 三人结构比二人结构更容易制造错位和混乱

---

## 三、起承转合四步结构（5分钟短剧黄金公式）

### 起（30秒）
- 建立人物 + 场景
- 让观众知道：谁、在哪、想干嘛

### 承（1分钟）
- 出现小冲突
- 角色的目标第一次受阻

### 转（2分钟）
- 冲突升级、人物被逼到极限
- **「目标→阻碍→再阻碍→彻底失控」的升级逻辑**
- 每一次升级都要叠加新的矛盾，不能重复

### 合（30秒）
- 反转 / 荒诞收尾
- **必须有明确结尾**——日式短剧拒绝开放式结局

---

## 四、节奏铁律

> **日式短剧拒绝冷场，节奏比剧情更重要**

### 基本节奏
- **15秒一个小笑点**
- **1分钟一个大爆点**

### 笑点藏在哪里
不只是对话，还有：
- 动作
- 表情
- 台词
- 反应

---

## 五、高级心法（三谷幸喜）

### 1. 喜剧的内核是「认真」
- 角色越认真，行为越荒诞，反差越大越好笑
- 哪怕演傻子，也要一本正经地傻

### 2. 日常解构法
- 把生活里的小情绪（焦虑、较真、讨好）放大10倍
- 笑点不是编出来的，是生活里本来就有的

### 3. 反讽与留白
- 不直接讲笑话，让观众自己品出笑点
- 高级短剧不靠密集梗，靠人物逻辑让观众笑

---

## 六、创作检查清单

### 角色设计
- [ ] 有一个明确的性格标签（偏执/懦弱/自大/较真）
- [ ] 性格贯穿始终，绝不崩塌
- [ ] 有一个明确的目标

### 情境设计
- [ ] 是日常场景（办公室、医院、便利店等）
- [ ] 场景被某个事件打破
- [ ] 日常场景 + 极端人物 = 自动冲突

### 结构检查
- [ ] 起：30秒内建立人物+场景
- [ ] 承：1分钟内出现第一次冲突
- [ ] 转：2分钟冲突升级，至少3次阻碍
- [ ] 合：30秒内有明确收尾（反转或荒诞结局）

### 节奏检查
- [ ] 每15秒至少一个小笑点
- [ ] 每1分钟有一个大爆点
- [ ] 笑点密度：5分钟 ≈ 20个笑点（≥4个/分钟）

---

## 七、常见错误

| 错误 | 说明 |
|------|------|
| 人设崩塌 | 角色为了笑点改变了性格，破坏可信度 |
| 无目标 | 角色只是在场景里"晃悠"，没有明确目的 |
| 重复阻碍 | 三次阻碍用同一个梗，观众会腻 |
| 冷场 | 超过30秒没有笑点 |
| 开放式结局 | 日式短剧必须有明确收尾 |

---

## 八、与 Sketch 的区别

| 维度 | 日式短剧（コント） | Sketch |
|------|------------------|--------|
| 节奏 | 极快，15秒必笑 | 可缓可急 |
| 人物 | 一个极端性格贯穿 | 可有多面性 |
| 场景 | 日常场景被打破 | 可荒诞设定 |
| 目标 | 明确单一目标 | 可有多目标 |
| 结尾 | 必须明确收尾 | 可开放式 |

日式短剧更像"把一个极端人物逼到极限"，Sketch更像"用一个设定展开故事"。
