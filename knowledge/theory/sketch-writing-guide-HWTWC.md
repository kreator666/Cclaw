## 第一章 什么是 Sketch

**定义**：Sketch（情景小短剧）往往从一个有趣的创意展开，情节飞速递进，最后夸张到极致，故事要么在高潮中戛然而止，要么最后有一个大反转。

**时长**：一般 3–5 分钟，包含一定的对话和动作，基本符合"三一律"原则——在当下的场景实时发生的故事，一般不包括前史。

**五大特点**：
1. **单一场景**：从头到尾不切换场景
2. **少量角色**：一般只在几个角色之间展开
3. **核心冲突**：一开始就交代清楚，无旁枝末节
4. **实时性**：舞台上两分钟等于真实两分钟，不倒叙不插叙
5. **核心角色特点不变**：正常人代表观众视角，提供反应，制造反差

**Sketch vs 中式小品**：

| | 中式小品 | Sketch |
|---|---|---|
| 结构 | 起承转合，完整故事 | 片段，无完整戏剧结构 |
| 重心 | 故事本身 | **创意和精妙设定** |
| 节奏 | 慢而自然，生活化 | 飞速递进，夸张突破想象 |
| 时长 | 较长 | 3–5 分钟 |

## 第二章 收集创意

### 思维导图扩展法

1. 选择一个主题，围绕主题构思相关元素（地点、人物、物品）
2. 尽量多想——通过量的积累激发灵感，**先不管好笑不好笑**
3. 尽量想大众熟知的元素，方便观众理解
4. 想到的元素要**富有延展性**，便于故事展开
5. 避开抽象模糊的单一元素（如"贪婪""黑暗"），去想具体的事、人或物

### 从创意到喜剧创意——四种喜剧技巧

#### 1. 混搭（Juxtaposition）
把两个本身无关、不相称的东西混搭在一起，产生喜剧效果。
- 典型技巧：**"鱼离水"**——让角色或事件离开熟悉的环境，到达不适的新环境，他与新环境的不协调产生笑点
  - 例：一个不善言辞的人，平时没问题（鱼在水里），但放在婚礼致辞现场（鱼离开水），他的"不适感"贯穿始终

#### 2. 夸张
将剧情里的元素遵循一定逻辑逐步夸张，触发笑点。可以不合常理，但故事中的逻辑要行得通。
- 例：帽子越来越小，最后小到需要显微镜才能看见

#### 3. 戏仿（Parody）
把概念或事件放在大家熟知的经典 IP 里（小说、历史典故、神话、童话、影视作品），强化其中不协调的部分。
- 例：逼婚发生在武侠小说里，逼婚对象变成行走江湖的大侠

#### 4. 翻转
将喜剧创意中的特点进行翻转，制造反差。
- 例：同性恋世界里的异性恋受歧视（现实世界的翻转）

### 创意筛选标准
- 喜剧创意必须包含**冲突**，冲突是让 Sketch 发展的动力
- 如果创意本身不含冲突，需要**制造**冲突

## 第三章 从喜剧创意到前提

### 什么是前提（Premise）
前提是使 Sketch 得以发展的**初始条件**，是关于故事逻辑或人物行为模式的基础设定，包含三要素：**人物、地点、事件**。

### 三要素建立

**人物**：
- 建立足够数量的人物来体现冲突
- "**疯子**"（有突出不正常特征的）+ "**正常人**"（代表观众视角）
- 建议 Sketch 中"疯子"不超过 1 个（熟练后可增加到 2 个）
- 人物性格要**直露、突出**，一般一个 sketch 只需要一个特点

**地点**：
- 选择最适合展现冲突发生的场景
- 让正常人无法轻易离开，使冲突可以延续

**事件**：
- 角色正在经历的事情，表明行为动机和目标
- 让 Sketch 有一个"动能"，自发地前进

### 前提检查清单
1. 能否用**一句话**解释清楚？
2. 是否有**延展空间**（能想到丰富的具体场景和行为）？
3. 能否在**几分钟内视觉化呈现**？（某些题材适合视频但不适合舞台）
4. 是否包含具体的**人物、场景、事件**？

> **核心原则**：前提只是初始条件，不是内容梗概或故事大纲。说出过程或结局的内容先删掉。

## 第四章 从前提到结构

### 三段式结构

| 阶段 | 名称 | 内容 |
|---|---|---|
| 设定 | The Setup | 最短时间内将前提告诉观众 |
| 发展 | The Build | 前提中事件的进一步发展，冲突逐渐夸张化 |
| 结尾 | The Ending | 强结尾，让笑点推到极致后戛然而止 |

### 设定阶段
- 需要在**十几秒**内让观众明白前提是什么
- 不需要展示全过程，直接从最能体现前提的那一刻开始
- 超自然设定也不用解释原因（解释只会增加没有笑点的冗余内容）

### 发展阶段——加笑点的方法

**列清单**（最重要）：
- 思考在具体环境下前提是如何体现的，用头脑风暴列出可能性
- 包括可能出现的对白、动作或事件

**将清单配对**（适用于混搭类）：
- 将两个元素的特点各自列出，然后配对，看哪些能产生喜剧效果

**夸张**：
- 想象一个夸张的人物会有哪些行为和语言

**重复**：
- 让人物的关键行为或关键语言重复，隔一段时间重复一次，引爆笑点

### 笑点排序与串联
- 按**时间逻辑**串联笑点（发生先后）
- 在每个时间段内，按**普通→极致**的夸张程度排序
- 删去雷同的笑点（观众有了预期后，类似的笑点无法超越预期）
- 笑点数量参考：两分钟 Sketch **至少需要 6 次大笑**

### 结尾六种方式
1. **呼应**：将之前的行为再重复一次作为结尾
2. **反转**：在结尾处改变之前的前提（或让正常人做出不正常的举动）
3. **极致的夸张**：无以复加的超自然夸张
4. **夸张的扩大**：从一个不正常的人/事件扩展到更大的范围（全世界都这样）
5. **大笑点**：一个很大的笑点把气氛推到极致时戛然而止
6. **自然结尾**：事件自然结束，不需要刻意设计

### 阶段区分（概念辨析）

| 阶段 | 说明 |
|---|---|
| 创意 | 启发词、素材、有意思的点 |
| 喜剧创意 | 经过喜剧技巧加工后的概念 |
| 前提 | 确定具体人物、场景、事件 |
| 结构 | Sketch 的梗概，包含主要笑点和节奏 |

## 第五章 Sketch 的写作和修改

### 对话写作要点
- **接近日常用语**：不要堆砌俏皮话
- **精炼**：一分钟要有三个笑点，不必要的词、重复的话、太长的话全部删去
- **情绪外露**：用台词体现角色此刻的情绪
- **考虑不说话角色的反应**：用表情、肢体创造更多笑点
- **角色语言模式一致**：口头禅、书面语、行业术语等
- **笑点后置**：把笑点放在对话的后面

### 修改检查表
- [ ] 是否按照前提写，有没有偏离？
- [ ] 开头几句话内是否交代出了前提？
- [ ] 长度控制在 2–3 分钟（两三页 A4 纸）？
- [ ] 角色数量不超过 4 个？
- [ ] 是否在同一个地点发生？
- [ ] 一次一次精简对话？（没有一个 Sketch 不能被精简 20 遍）
- [ ] 两分钟的 Sketch 需要至少 6 次大笑
- [ ] 结尾是否有很大的笑点？

## 第六章 角色 Sketch 的写作

### 角色 Sketch 要点
- 整个 Sketch 围绕一个有鲜明特点的人物展开（如憨豆先生）
- 角色特点要**非常鲜明，卡通化**，让观众很快理解
- 特点要**外露**，内在性格也要通过外部语言和动作表现
- 角色需要有**具体的动机**（不是单纯展示特点）
- 角色要有**标致性的语言特征或口头禅**
- 特点要**贯穿始终**（电影里人会变，Sketch 里人不变）

### 为角色配配角
- 配角用来**推动、激发、引诱**主角展现特点
- 可以是无意中让主角发作，或有意地让主角升级
- 配角一般是"正常人"，衬托主角的荒谬

### 鱼离水法找场景
- 让角色离开熟悉的环境，到达不适的新环境
- 这样角色会显得格格不入，冲突自然发生

## 附录：现场 Sketch 创作流程

```
①收集创意 → ②确定前提 → ③确定结构大纲及笑点
→ ④形成初稿 → ⑤读稿与修改反馈（不断重复）
→ ⑥演员排练 → ⑦演出 → ⑧观看演出视频，继续调整
```

### 编剧会注意事项
- 参与人数以 **4 人以内**为宜
- 主编负责带领方向，编剧助理负责完整记录
- 创意阶段**多认同少批评**，多在他人创意基础上增加改善
- 如果 5 分钟以上思维卡壳，及时换素材或思路
- 编剧**不要有执念或控制欲**，死磕自己的想法不听他人发言是创作大忌

## 核心框架速查

```
创意 → 喜剧创意（四技巧：混搭/夸张/戏仿/翻转）
  → 前提（人物+地点+事件）
    → 结构（设定→发展→结尾）
      → 剧本写作与修改
```

**三段式结构核心**：
- 设定：快速、清晰告诉观众前提
- 发展：两到三件事，将冲突逐渐升级（每个事件 2–3 个笑点）
- 结尾：强结尾（6 种结尾方式可选）

**人物核心**：
- 疯子（特点鲜明，贯穿始终）+ 正常人（代表观众，制造反差）
- 建议不超过 1 个疯子（熟练后可 2 个）
