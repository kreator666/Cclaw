# 脱口秀输出模板

> 适用：单人口述喜剧，5-20分钟。核心是"我"的视角 + 铺垫-反转节奏。

## 输出规格

### 结构规范

```
【脱口秀标题】

【开场 Hook】（钩子，30秒内抓住注意力）

【第一段：建立世界观】
铺垫内容 → 核心笑点
（2-3个铺垫-反转段落）

【第二段：深挖细节】
更多细节 → 更深层的笑点
（call back 到第一段的强梗——时机：观众快忘掉时）

【脆弱时刻】（可选，1处）
制造共鸣的真诚段落

【第三段：疯狂升级】
笑点密度加大，节奏加快
（3-4个快节奏段子）

【收尾：炸裂 Call Back】
遗忘后再炸——把中前段的强梗在中后段再次激活，笑点极强
```

## 创作步骤

### Step 1：确定核心主题句

一句话说清你想讲什么。问自己：这里有什么"生命的机械化"？

### Step 2：找三个故事/观察

脱口秀最好有具体细节。三个故事/观察点，形成递进。

### Step 3：设计铺垫-反转结构

```
铺垫 = 让观众往 A 方向想
笑点 = 揭示其实是 B 方向

铺垫要"真"——观众必须相信才能笑
反转要"快"——不要铺垫太久才反转
```

### Step 4：设计 Call Back

找一个在前半段已经验证过（观众笑过）的强梗，在中后段某处再次使用——时机是观众差不多忘掉了。
不要"呼应开头"，要"遗忘后再炸"。

如果前半段没有强梗，先把梗讲扎实，再考虑 Call Back。

## 手法清单（脱口秀优先）

| 手法 | 说明 |
|------|------|
|铺垫-反转|核心结构|
|Call back|遗忘后再炸——在观众差不多忘掉一个强梗时，突然再次使用，笑点翻倍|
|身份错位|我是X但我做了Y|
|逻辑滑坡|从一个正常前提推到荒谬结论|
|自我打击|先说一个观点然后自己反驳|
|细节堆叠|用超具体的细节制造荒谬|
|反向预期|观众以为你会说A，你说了B|

### Call Back 的正确理解

> Call Back 不是呼应，不是平行，不是回归。
> Call Back 是一个强梗，在观众已经遗忘它的时候，突然再次杀出——炸。

**关键要素：**
- **时机**：观众差不多忘掉了，不是刚讲完就回
- **效果**：再次触发 = 比第一次更爆，因为有"遗忘+突然唤醒"的反差
- **次数**：一个梗 Call Back 一次就够了，两次就啰嗦

**常见误区：**
- ❌ "开头用了一个梗，结尾呼应一下"——这是平行，不是 Call Back
- ❌ "在同一个段落里重复使用同一个梗"——这是堆叠，不是 Call Back

**结构里"收尾 Call Back"的真实含义：**
不是呼应开头，而是——
在整段脱口秀的后半段，找一个前面埋下的强梗，在观众快忘掉时重新激活。
如果那个强梗恰好和开头有关，叫它"首尾呼应"也无妨，但那只是时机巧合，不是本质。

## 笑点密度控制

```
密集型：每30-45秒一个笑点
标准型：每60-90秒一个笑点  
稀疏型：笑点作为段落高潮点缀
```
